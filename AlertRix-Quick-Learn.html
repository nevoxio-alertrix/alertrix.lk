Interview Quick Learn

AlertRix Repository Summary

A React/Vite product showcase website for an AI-powered medication management companion.

1. Project Purpose

AlertRix is presented as an AI-powered medication management device for seniors, caregivers, and families. The website explains the product vision, shows how the device works, highlights key features, and guides users toward preordering, joining a waitlist, or contacting the team.

Interview line: "This is a React-based product website for AlertRix, an AI medication-care companion that helps seniors avoid missed doses through voice reminders, guardian alerts, scheduling, and AI-assisted monitoring."

2. Technology Stack

Frontend
  • React 18
  • Vite
  • React Router DOM
  • CSS Modules
Motion / UX
  • GSAP and ScrollTrigger
  • @gsap/react
  • Anime.js
  • Lenis smooth scrolling
Deployment
  • Vercel configuration
  • SPA rewrites to index.html
  • Production build via vite build
Assets
  • Product images
  • Explainer videos
  • Logo and team visuals

3. Main Routes

  • / - Home page with hero, product overview, stats, gallery, app preview, and preorder CTA.
  • /product - Product problem statement, videos, pricing cards, and waitlist UI.
  • /features - Feature walkthrough and capability cards.
  • /about - Mission, vision, values, team members, and contact links.

4. Key Functionalities

  • Responsive navigation with desktop links and mobile drawer.
  • Smooth page scrolling on fine-pointer devices.
  • Custom animated cursor for desktop users.
  • Scroll-triggered reveal animations across sections.
  • Hero parallax effects and animated SVG line art.
  • Product videos with posters and autoplay/loop behavior.
  • Pricing cards for Standard and Premium plans.
  • Footer with navigation, Instagram, WhatsApp, LinkedIn, and enquiry details.

5. Product Features Presented

  • Smart voice alerts for medication reminders.
  • AI chatbot companion for health queries and check-ins.
  • Guardian alerts when doses are missed or emergencies happen.
  • Smart scheduling that adapts to routines.
  • Dual power system with backup battery.
  • Future-ready OTA updates.
  • Health summaries, cloud uploads, and priority alerts in Premium.

6. Architecture Summary

The application is a client-side single-page app. src/App.jsx registers routes, adds the navbar, enables smooth scrolling, resets scroll position on route changes, and conditionally renders the custom cursor. Each page has its own JSX file and CSS Module. Shared UI lives in src/components, while reusable Anime.js hooks live in src/hooks/useAnime.js.

7. Current Limitations

  • No backend, database, authentication, payment system, or admin dashboard.
  • The waitlist form prevents default submission but does not send or store emails.
  • There is a duplicated nested project folder named alertrix.lk, and some duplicate files differ.
  • README is minimal, so project documentation could be improved.

8. Interview Pitch

"AlertRix is a Vite and React single-page application built as a polished product website for an AI-powered medication management device. I used React Router for navigation, CSS Modules for scoped styling, and GSAP, Anime.js, and Lenis to create a smooth interactive experience. The site presents the product story, features, pricing, preorder flow, videos, gallery, team, and contact channels. It is currently frontend-focused, so future improvements would include connecting the waitlist to a backend, adding analytics, improving accessibility, and integrating real preorder or payment functionality."
